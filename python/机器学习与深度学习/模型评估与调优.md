# 模型评估与调优

## What — 模型评估与调优是什么

模型评估是用恰当的指标和方法衡量模型泛化能力，模型调优是通过优化超参数和训练策略提升模型性能。两者紧密关联：评估指导调优方向，调优后需重新评估。不严谨的评估会导致"纸上好看、线上翻车"。

核心问题：

| 问题 | 说明 |
|------|------|
| 过拟合 vs 欠拟合 | 训练集好但验证集差 vs 两者都差 |
| 评估指标选择 | 不同任务用不同指标 |
| 数据泄露 | 评估时无意使用了未来信息 |
| 超参搜索效率 | 穷举太慢，随机不够最优 |

## Why — 为什么评估与调优如此关键

模型训练只是机器学习项目的一部分。一个训练集上 99% 准确率的模型可能因为过拟合在真实场景只有 60%。严谨的评估和系统化的调优是确保模型可用性的必要条件——这往往是工业项目和学术实验之间的最大差距。

## How — 评估与调优方法

### 1. 分类评估指标

```python
import numpy as np
from sklearn.metrics import (
    accuracy_score, precision_score, recall_score, f1_score,
    classification_report, confusion_matrix,
    roc_auc_score, roc_curve, precision_recall_curve,
    average_precision_score, log_loss, cohen_kappa_score, matthews_corrcoef
)
import matplotlib.pyplot as plt

rng = np.random.default_rng(42)
y_true = rng.choice([0, 1], 200, p=[0.7, 0.3])
y_prob = np.clip(y_true * 0.8 + rng.normal(0, 0.3, 200), 0, 1)
y_pred = (y_prob >= 0.5).astype(int)

# === 基础指标 ===
print(f"准确率 (Accuracy):  {accuracy_score(y_true, y_pred):.4f}")
print(f"精确率 (Precision): {precision_score(y_true, y_pred):.4f}")
print(f"召回率 (Recall):    {recall_score(y_true, y_pred):.4f}")
print(f"F1 Score:           {f1_score(y_true, y_pred):.4f}")

# === 混淆矩阵 ===
cm = confusion_matrix(y_true, y_pred)
print(f"\n混淆矩阵:\n{cm}")
#              预测负  预测正
# 实际负  [  TN      FP  ]
# 实际正  [  FN      TP  ]

# === 详细报告 ===
print(f"\n{classification_report(y_true, y_pred, target_names=['负类', '正类'])}")

# === ROC 曲线与 AUC ===
fpr, tpr, thresholds_roc = roc_curve(y_true, y_prob)
auc = roc_auc_score(y_true, y_prob)

# === PR 曲线 ===
precisions, recalls, thresholds_pr = precision_recall_curve(y_true, y_prob)
ap = average_precision_score(y_true, y_prob)

fig, axes = plt.subplots(1, 2, figsize=(12, 5))

axes[0].plot(fpr, tpr, label=f'ROC AUC={auc:.3f}')
axes[0].plot([0, 1], [0, 1], 'k--')
axes[0].set_xlabel('FPR')
axes[0].set_ylabel('TPR')
axes[0].set_title('ROC 曲线')
axes[0].legend()

axes[1].plot(recalls, precisions, label=f'PR AP={ap:.3f}')
axes[1].set_xlabel('Recall')
axes[1].set_ylabel('Precision')
axes[1].set_title('PR 曲线')
axes[1].legend()

plt.tight_layout()
plt.show()

# === 其他指标 ===
print(f"Log Loss: {log_loss(y_true, y_prob):.4f}")
print(f"Cohen's Kappa: {cohen_kappa_score(y_true, y_pred):.4f}")
print(f"MCC: {matthews_corrcoef(y_true, y_pred):.4f}")

# === 多分类指标 ===
y_true_multi = rng.integers(0, 5, 200)
y_pred_multi = rng.integers(0, 5, 200)

print(f"\n多分类 F1 (macro): {f1_score(y_true_multi, y_pred_multi, average='macro'):.4f}")
print(f"多分类 F1 (micro): {f1_score(y_true_multi, y_pred_multi, average='micro'):.4f}")
print(f"多分类 F1 (weighted): {f1_score(y_true_multi, y_pred_multi, average='weighted'):.4f}")
```

| 指标 | 公式 | 关注点 | 适用场景 |
|------|------|--------|---------|
| Accuracy | (TP+TN)/N | 整体正确 | 类别均衡 |
| Precision | TP/(TP+FP) | 预测正的有多少真 | 误报代价高 |
| Recall | TP/(TP+FN) | 真正的有多少被找到 | 漏报代价高 |
| F1 | 2PR/(P+R) | 精确和召回平衡 | 不平衡分类 |
| AUC-ROC | ROC曲线下面积 | 排序能力 | 类别不太极端不平衡 |
| AP | PR曲线下面积 | 正类排序能力 | 极端不平衡 |

### 2. 回归评估指标

```python
import numpy as np

print(f"Max Error: {max_error(y_true, y_pred):.4f}")

residuals = y_true - y_pred

fig, axes = plt.subplots(1, 2, figsize=(12, 5))

axes[0].axhline(y=0, color='red', linestyle='--')
axes[0].set_xlabel('预测值')
axes[0].set_ylabel('残差')
axes[0].set_title('残差图')

axes[1].hist(residuals, bins=30, edgecolor='white')
axes[1].set_title('残差分布')

plt.show()
```

| 指标 | 说明 | 对离群值 | 量纲 |
|------|------|---------|------|
| MSE | 均方误差 | 敏感（平方放大） | 原始² |
| RMSE | 均方根误差 | 敏感 | 原始 |
| MAE | 平均绝对误差 | 较鲁棒 | 原始 |
| R² | 解释方差比例 | — | 无量纲 |
| MAPE | 百分比误差 | y≈0时爆炸 | % |
| MedAE | 中位绝对误差 | 鲁棒 | 原始 |

### 3. 过拟合诊断

```python
import numpy as np
from sklearn.model_selection import learning_curve, validation_curve
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import make_classification
import matplotlib.pyplot as plt

X, y = make_classification(n_samples=1000, n_features=20, random_state=42)

# === 学习曲线 ===
train_sizes, train_scores, val_scores = learning_curve(
    RandomForestClassifier(n_estimators=100, random_state=42),
    X, y, cv=5, train_sizes=np.linspace(0.1, 1.0, 10),
    scoring='accuracy', n_jobs=-1
)

fig, ax = plt.subplots(figsize=(10, 6))
ax.plot(train_sizes, train_scores.mean(axis=1), 'o-', label='训练集')
ax.fill_between(train_sizes,
                train_scores.mean(axis=1) - train_scores.std(axis=1),
                train_scores.mean(axis=1) + train_scores.std(axis=1), alpha=0.1)
ax.plot(train_sizes, val_scores.mean(axis=1), 'o-', label='验证集')
ax.fill_between(train_sizes,
                val_scores.mean(axis=1) - val_scores.std(axis=1),
                val_scores.mean(axis=1) + val_scores.std(axis=1), alpha=0.1)
ax.set_xlabel('训练样本数')
ax.set_ylabel('准确率')
ax.set_title('学习曲线')
ax.legend()
plt.show()

# 诊断：
# 训练高 + 验证低 → 过拟合（需要正则化/更多数据/简化模型）
# 训练低 + 验证低 → 欠拟合（需要更复杂模型/更多特征）
# 两者接近但都低 → 需要更多数据或更好的特征
# 两者接近且高 → 理想状态

# === 验证曲线 ===
from sklearn.svm import SVC
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline

pipe = Pipeline([('scaler', StandardScaler()), ('svc', SVC())])

param_range = np.logspace(-3, 2, 6)
train_scores, val_scores = validation_curve(
    pipe, X, y, param_name='svc__C', param_range=param_range,
    cv=5, scoring='accuracy', n_jobs=-1
)

fig, ax = plt.subplots(figsize=(8, 5))
ax.semilogx(param_range, train_scores.mean(axis=1), 'o-', label='训练集')
ax.semilogx(param_range, val_scores.mean(axis=1), 'o-', label='验证集')
ax.set_xlabel('C (正则化强度)')
ax.set_ylabel('准确率')
ax.set_title('验证曲线')
ax.legend()
plt.show()
```

### 4. 超参调优进阶

```python
import numpy as np
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split, GridSearchCV, RandomizedSearchCV
from sklearn.ensemble import RandomForestClassifier
from scipy.stats import loguniform

# === HalvingGridSearchCV（渐进式搜索，更高效） ===
from sklearn.experimental import enable_halving_search_cv
from sklearn.model_selection import HalvingGridSearchCV

param_grid = {
    'n_estimators': [50, 100, 200],
    'max_depth': [3, 5, 10, None],
    'min_samples_split': [2, 5, 10],
}

halving = HalvingGridSearchCV(
    RandomForestClassifier(random_state=42),
    param_grid, cv=5, factor=3,  # 每轮淘汰 2/3
    scoring='accuracy', n_jobs=-1, random_state=42
)
halving.fit(X_train, y_train)
print(f"Halving 最佳: {halving.best_params_}")
print(f"迭代次数: {halving.n_iterations_}")

# === Optuna（贝叶斯优化，最推荐） ===
import optuna

def objective(trial):
    n_estimators = trial.suggest_int('n_estimators', 50, 300)
    max_depth = trial.suggest_int('max_depth', 3, 20)
    min_samples_split = trial.suggest_int('min_samples_split', 2, 20)
    max_features = trial.suggest_categorical('max_features', ['sqrt', 'log2', None])

    clf = RandomForestClassifier(
        n_estimators=n_estimators, max_depth=max_depth,
        min_samples_split=min_samples_split, max_features=max_features,
        random_state=42
    )
    from sklearn.model_selection import cross_val_score
    scores = cross_val_score(clf, X_train, y_train, cv=5, scoring='accuracy')
    return scores.mean()

study = optuna.create_study(direction='maximize')
study.optimize(objective, n_trials=50, show_progress_bar=True)

print(f"最佳参数: {study.best_params}")
print(f"最佳分数: {study.best_value:.4f}")
```

### 6. 交叉验证进阶

```python
from sklearn.model_selection import (
    cross_val_score, cross_validate, StratifiedKFold,
    RepeatedStratifiedKFold, GroupKFold
)
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import make_classification
import numpy as np

X, y = make_classification(n_samples=500, random_state=42)

# === cross_validate（多指标+时间） ===
cv_results = cross_validate(
    RandomForestClassifier(n_estimators=100, random_state=42),
    X, y, cv=5,
    scoring=['accuracy', 'f1', 'roc_auc'],
    return_train_score=True,
    n_jobs=-1
)

print("训练准确率:", cv_results['train_accuracy'].mean(), "±", cv_results['train_accuracy'].std())
print("验证准确率:", cv_results['test_accuracy'].mean(), "±", cv_results['test_accuracy'].std())
print("验证F1:", cv_results['test_f1'].mean())
print("验证AUC:", cv_results['test_roc_auc'].mean())
print("拟合时间:", cv_results['fit_time'].sum(), "秒")

# === 重复分层KFold（更稳定） ===
rskf = RepeatedStratifiedKFold(n_splits=5, n_repeats=10, random_state=42)
scores = cross_val_score(
    RandomForestClassifier(n_estimators=100, random_state=42),
    X, y, cv=rskf, scoring='accuracy', n_jobs=-1
)
print(f"\n重复10次5折CV: {scores.mean():.4f} ± {scores.std():.4f}")

# === GroupKFold（防止数据泄露） ===
groups = np.repeat(np.arange(100), 5)  # 100组，每组5个样本
gkf = GroupKFold(n_splits=5)
scores = cross_val_score(
    RandomForestClassifier(n_estimators=100, random_state=42),
    X, y, cv=gkf, groups=groups, scoring='accuracy'
)

# === 嵌套交叉验证（无偏估计） ===
from sklearn.model_selection import GridSearchCV

inner_cv = StratifiedKFold(n_splits=3, shuffle=True, random_state=42)
outer_cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

clf = GridSearchCV(
    RandomForestClassifier(random_state=42),
    {'max_depth': [3, 5, 10, None], 'n_estimators': [50, 100]},
    cv=inner_cv, scoring='accuracy'
)

nested_scores = cross_val_score(clf, X, y, cv=outer_cv, scoring='accuracy')
print(f"\n嵌套CV无偏估计: {nested_scores.mean():.4f} ± {nested_scores.std():.4f}")
```

### 7. 模型校准

模型输出的概率是否准确反映了真实概率？很多模型（如SVM、随机森林）的原始概率不可靠，需要校准。

```python
from sklearn.calibration import CalibratedClassifierCV, calibration_curve
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split
import matplotlib.pyplot as plt

X, y = make_classification(n_samples=2000, n_features=20, random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# 未校准的随机森林
rf = RandomForestClassifier(n_estimators=100, random_state=42)
rf.fit(X_train, y_train)

# Platt Scaling（sigmoid 校准）
rf_platt = CalibratedClassifierCV(rf, method='sigmoid', cv=5)
rf_platt.fit(X_train, y_train)

# Isotonic Regression（非参数校准）
rf_iso = CalibratedClassifierCV(rf, method='isotonic', cv=5)
rf_iso.fit(X_train, y_train)

# 可视化校准曲线
fig, ax = plt.subplots(figsize=(8, 6))
for name, model in [('未校准', rf), ('Platt', rf_platt), ('Isotonic', rf_iso)]:
    prob_pos = model.predict_proba(X_test)[:, 1]
    fraction_pos, mean_pred = calibration_curve(y_test, prob_pos, n_bins=10)
    ax.plot(mean_pred, fraction_pos, 'o-', label=name)

ax.plot([0, 1], [0, 1], 'k--', label='完美校准')
ax.set_xlabel('预测概率')
ax.set_ylabel('实际正类比例')
ax.set_title('校准曲线')
ax.legend()
plt.show()
```

## 面试题

### 1. 为什么不能只用准确率评估分类模型？

准确率在类别不平衡时会产生误导。例如 95% 负样本的数据集，全猜负也有 95% 准确率，但模型完全无法识别正类。应该根据业务目标选择指标：关注误报代价用 Precision，关注漏报代价用 Recall，需要平衡用 F1。极端不平衡时用 PR-AUC 比 ROC-AUC 更可靠——ROC-AUC 在负类远多于正类时过于乐观。多分类时 macro-F1 对每个类平等对待，weighted-F1 按类别大小加权。

### 2. 交叉验证和 train/val/test 划分的关系是什么？

最佳实践是两者结合：用交叉验证在训练集上做模型选择和超参调优，保留一个独立的测试集做最终评估。交叉验证用于比较不同模型/参数，给出泛化性能的稳健估计；测试集只用一次，给出无偏的最终评估。常见错误：在全部数据上做交叉验证选参数，然后报告CV分数——这会高估性能，因为调参过程"看到了"所有数据。嵌套交叉验证可以给出完全无偏的估计，但计算量大。

### 3. GridSearchCV 调参为什么容易过拟合验证集？

每次调参都在验证集上看效果，相当于在验证集上"训练"超参数——调了100次参数，即使随机波动也可能出现高分的组合。缓解方法：（1）使用交叉验证而非单次划分；（2）用嵌套交叉验证获得无偏估计；（3）扩大验证集；（4）减少搜索空间，避免盲目穷搜；（5）最终报告必须在独立测试集上，而非CV最佳分数。Optuna 的贝叶斯优化利用历史信息指导搜索，比网格搜索更高效，但同样存在这个问题。

### 4. L1 和 L2 正则化的区别？

L1（Lasso）在损失函数中加入权重的绝对值之和，倾向于产生稀疏解（部分权重精确为0），天然具有特征选择效果。L2（Ridge）加入权重的平方和，倾向于将权重压缩到较小值但不为零，更适合特征间有共线性的情况。几何解释：L1 约束区域是菱形，等高线更容易在顶点相交（权重为0的轴）；L2 约束区域是圆形，交点通常不在轴上。实践中，不确定用哪个时选 ElasticNet（L1+L2 混合），深度学习用 L2（权重衰减）。

### 5. 什么是偏差-方差权衡？

偏差（Bias）是模型预测值与真实值的系统性偏差，高偏差意味着欠拟合（模型太简单学不到模式）。方差（Variance）是模型对训练数据变化的敏感度，高方差意味着过拟合（模型学了噪声）。总误差 = 偏差² + 方差 + 不可约误差。增加模型复杂度降低偏差但增大方差，反之亦然。深度学习中通常不用担心偏差（模型足够复杂），重点是降低方差（正则化、数据增强、早停）。集成方法（Bagging 降方差，Boosting 降偏差）是另一种解法。

### 6. Early Stopping 的原理和注意事项？

Early Stopping 在验证集性能不再提升时停止训练，是防止过拟合最实用的方法。原理：训练初期模型学到有用的模式（偏差下降），后期开始拟合噪声（方差上升），在验证误差最低点停止。注意事项：（1）patience 不能太小（可能太早停止），也不能太大（过拟合太严重），通常 5-10 个 epoch；（2）必须 restore_best_weights=True，否则用的是停止时的权重而非最佳权重；（3）验证集必须与训练集独立，不能有数据泄露；（4）学习率调度要配合——如果 LR 突然下降，验证损失可能短暂上升后继续下降，过早停止会错过。

### 7. 如何判断模型是否需要校准？

如果模型输出的概率与真实概率不一致，就需要校准。例如模型预测概率 0.8，但实际只有 60% 是正类。判断方法：（1）画校准曲线（calibration curve），偏离对角线则需要校准；（2）计算 Brier Score（概率预测的 MSE），越低越好。通常需要校准的模型：SVM、决策树、随机森林（投票概率不连续）、朴素贝叶斯。通常不需要校准的模型：逻辑回归（天然输出校准概率）、梯度提升树。校准方法：Platt Scaling（sigmoid 拟合，适合小数据）和 Isotonic Regression（非参数，需要较多数据）。

### 8. 嵌套交叉验证的作用是什么？

嵌套交叉验证用于给出模型选择+超参调优后泛化性能的无偏估计。外层CV做性能评估，内层CV做模型选择。外层每个fold的测试集从未参与任何决策（包括参数选择），因此估计是无偏的。简单CV用同一折做参数选择和性能评估会过于乐观——因为参数是在验证集上选的，再在同一个验证集上评估等于"自己考自己"。嵌套CV计算量是外折×内折×参数组合，但给出了最可靠的性能估计，适合论文和严谨的评估。
