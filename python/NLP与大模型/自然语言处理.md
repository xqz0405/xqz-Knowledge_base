# 自然语言处理

## What — NLP 是什么

自然语言处理（NLP）是让计算机理解、生成和处理人类语言的技术领域。从文本分类、情感分析到机器翻译、对话系统，NLP 覆盖了语言相关的各类任务。深度学习时代，NLP 的范式从统计方法 → 词嵌入 → RNN/LSTM → [[深度学习架构#3. Transformer 与自注意力|Transformer]] 快速演进，当前以预训练大模型为主导。

核心任务分类：

| 任务类型 | 说明 | 典型应用 |
|----------|------|---------|
| 文本分类 | 给文本分配类别 | 情感分析、垃圾检测 |
| 序列标注 | 逐 token 分类 | 命名实体识别、词性标注 |
| 文本生成 | 生成新文本 | 对话、摘要、翻译 |
| 关系抽取 | 提取实体关系 | 知识图谱构建 |
| 问答系统 | 根据问题生成答案 | 智能客服、搜索 |
| 文本匹配 | 判断文本相似度 | 语义搜索、去重 |

```python
# NLP 基本流程
raw_text → 分词 → 编码 → 模型 → 输出 → 解码
```

## Why — 为什么 NLP 重要

语言是人类最重要的信息载体，互联网上 80%+ 的数据是非结构化文本。NLP 让这些数据变得可计算——从社交媒体中挖掘舆情、从法律文档中提取条款、从客服对话中自动化应答。[[大模型与LLM应用]] 的爆发更让 NLP 成为 AI 应用落地的最前沿。

### 1. 分词（Tokenization）

```python

text = "Hello, world! This is NLP."

import nltk

# spaCy 分词（工业级，推荐）
```

### 2. 词嵌入（Word Embedding）

将离散的词映射为连续向量，使语义相近的词在向量空间中接近。

```python
import numpy as np

# === One-Hot 编码（最基础） ===
vocab = ['猫', '狗', '鱼', '鸟']
one_hot = {word: np.eye(len(vocab))[i] for i, word in enumerate(vocab)}
print(one_hot['猫'])  # [1, 0, 0, 0]

# === Word2Vec ===
from gensim.models import Word2Vec

# 训练 Word2Vec
sentences = [
    ['我', '喜欢', '猫'],
    ['我', '喜欢', '狗'],
    ['猫', '和', '狗', '是', '宠物'],
    ['鱼', '是', '动物'],
]
model_w2v = Word2Vec(sentences, vector_size=100, window=5, min_count=1, workers=4, sg=1)

# 获取词向量
vec_cat = model_w2v.wv['猫']
print(f"'猫' 向量维度: {vec_cat.shape}")

# 相似词
print(f"与'猫'最相似: {model_w2v.wv.most_similar('猫')}")

# 类比推理: 国王 - 男人 + 女人 = 女王
# result = model_w2v.wv.most_similar(positive=['国王', '女人'], negative=['男人'])

# === GloVe ===
# 加载预训练 GloVe 向量
# from gensim.models import KeyedVectors
# glove = KeyedVectors.load_word2vec_format('glove.6B.300d.txt', no_header=True)

# === FastText（子词嵌入，处理OOV） ===
from gensim.models import FastText
model_ft = FastText(sentences, vector_size=100, window=5, min_count=1)

# FastText 可以生成未登录词的向量（基于子词组合）
vec_unknown = model_ft.wv['猫狗']  # 即使没见过也能生成
print(f"OOV向量: {vec_unknown.shape}")

# === 上下文嵌入（ELMo/BERT） ===
from transformers import AutoModel, AutoTokenizer
import torch

tokenizer = AutoTokenizer.from_pretrained('bert-base-uncased')
model = AutoModel.from_pretrained('bert-base-uncased')

# "bank" 在不同上下文有不同的嵌入
texts = [
    "I went to the bank to deposit money.",     # 银行
    "The river bank was covered with flowers.",  # 河岸
]

for text in texts:
    inputs = tokenizer(text, return_tensors='pt')
    with torch.no_grad():
        outputs = model(**inputs)
    # 取最后一层隐状态作为上下文嵌入
    last_hidden = outputs.last_hidden_state
    # "bank" 的位置（不同句子中位置不同）
    bank_idx = (inputs['input_ids'] == tokenizer.convert_tokens_to_ids('bank')).nonzero()
    if len(bank_idx) > 0:
        bank_vec = last_hidden[bank_idx[0][0], bank_idx[0][1]]
        print(f"'{text[:30]}...' bank向量范数: {bank_vec.norm():.4f}")
```

| 嵌入类型 | 上下文相关 | OOV处理 | 代表模型 |
|----------|-----------|---------|---------|
| One-Hot | 否 | 否 | — |
| Word2Vec | 否 | 否 | Skip-gram/CBOW |
| GloVe | 否 | 否 | 全局共现矩阵 |
| FastText | 否 | 是（子词） | 子词n-gram |
| ELMo | 是 | 有限 | Bi-LSTM |
| BERT | 是 | 是（子词） | Transformer |

### 3. 文本分类

```python
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, Dataset
from transformers import AutoTokenizer, AutoModel

# === 传统方法：TF-IDF + 朴素贝叶斯 ===
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.naive_bayes import MultinomialNB
from sklearn.pipeline import Pipeline

texts = ["这部电影太精彩了", "服务很差，不推荐", "质量一般般", "非常满意"]
labels = [1, 0, 0, 1]

pipe = Pipeline([
    ('tfidf', TfidfVectorizer(max_features=5000)),
    ('clf', MultinomialNB())
])
pipe.fit(texts, labels)
print(f"预测: {pipe.predict(['太棒了'])}")  # [1]

# === 深度学习方法：LSTM ===
class TextClassifierLSTM(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim, num_classes):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.lstm = nn.LSTM(embed_dim, hidden_dim, batch_first=True, bidirectional=True)
        self.fc = nn.Linear(hidden_dim * 2, num_classes)
        self.dropout = nn.Dropout(0.3)

    def forward(self, x):
        emb = self.embedding(x)
        lstm_out, _ = self.lstm(emb)
        out = lstm_out[:, -1, :]
        return self.fc(self.dropout(out))

# === 预训练模型微调：BERT ===
class BertClassifier(nn.Module):
    def __init__(self, model_name='bert-base-chinese', num_classes=2, dropout=0.1):
        super().__init__()
        self.bert = AutoModel.from_pretrained(model_name)
        self.dropout = nn.Dropout(dropout)
        self.classifier = nn.Linear(self.bert.config.hidden_size, num_classes)

    def forward(self, input_ids, attention_mask):
        outputs = self.bert(input_ids=input_ids, attention_mask=attention_mask)
        cls_output = outputs.last_hidden_state[:, 0, :]  # [CLS] token
        return self.classifier(self.dropout(cls_output))

# 使用 Hugging Face Trainer 简化训练
from transformers import TrainingArguments, Trainer
from datasets import Dataset

# 准备数据
tokenizer = AutoTokenizer.from_pretrained('bert-base-chinese')

def tokenize_func(examples):
    return tokenizer(examples['text'], padding='max_length', truncation=True, max_length=128)

train_dataset = Dataset.from_dict({'text': texts, 'label': labels})
train_dataset = train_dataset.map(tokenize_func, batched=True)

# Trainer 训练
from transformers import AutoModelForSequenceClassification

model = AutoModelForSequenceClassification.from_pretrained(
    'bert-base-chinese', num_labels=2
)

training_args = TrainingArguments(
    output_dir='./results',
    num_train_epochs=3,
    per_device_train_batch_size=16,
    learning_rate=2e-5,
    weight_decay=0.01,
    evaluation_strategy='no',
)

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=train_dataset,
)
```

```python
# === 困惑度（语言模型） ===
import math
import torch

def perplexity(model, dataloader, tokenizer, device):
    model.eval()
    total_loss = 0
    total_tokens = 0
    with torch.no_grad():
        for batch in dataloader:
            input_ids = batch['input_ids'].to(device)
            labels = input_ids.clone()
            outputs = model(input_ids, labels=labels)
            total_loss += outputs.loss.item() * input_ids.size(1)
            total_tokens += input_ids.size(1)
    return math.exp(total_loss / total_tokens)
```

## 面试题

### 1. Word2Vec 的两种模型有什么区别？

Skip-gram 给定中心词预测上下文词，CBOW 给定上下文词预测中心词。Skip-gram 对每个中心词-上下文对生成一个训练样本，低频词得到更多训练，适合大数据集，向量质量更高但训练慢。CBOW 用上下文平均预测中心词，训练快但低频词表示较差。工业实践：小数据用 CBOW 快速得到可用向量，大数据用 Skip-gram 获得更精细的表示。两者都通过负采样（Negative Sampling）或层级 Softmax 加速训练。

### 2. BERT 的 MLM 和 NSP 分别是什么？

MLM（Masked Language Model）随机遮蔽 15% 的输入 token，让模型预测被遮蔽的词，是 BERT 的核心预训练任务，使模型获得双向上下文理解能力。NSP（Next Sentence Prediction）判断两个句子是否是原文中相邻的句子，让模型学习句子间的关系。后续研究（RoBERTa、ALBERT）发现 NSP 对大多数下游任务帮助不大，甚至有负面影响（因为随机配对的负样本太容易区分），因此很多改进模型移除了 NSP。

### 3. 为什么 Transformer 在 NLP 中取代了 RNN？

三个核心原因：（1）**并行化**——RNN 必须逐步处理，第 t 步依赖第 t-1 步的输出，无法并行；Transformer 的自注意力同时计算所有位置，充分利用 GPU；（2）**长距离依赖**——RNN 的信息需要逐步传递，超过数十步就严重衰减；Transformer 的自注意力让任意两个位置直接交互，路径长度 O(1)；（3）**可扩展性**——Transformer 架构可以方便地通过增加层数和宽度扩展到数十亿参数，RNN 的深度受梯度消失限制。代价是 O(n²) 复杂度，但各种稀疏注意力方案在缓解这个问题。

### 4. BPE 分词的原理和优势？

BPE 从字符级开始，迭代地合并最高频的字符对，直到达到目标词表大小。例如 "low" 出现5次 → "l"+"o" 合并为 "lo" → "lo"+"w" 合并为 "low"。优势：（1）词表大小可控（通常30k-50k），远小于词级词表（几十万到百万）；（2）能处理未登录词——任何词都可以拆为子词序列，如 "unhappiness" → "un", "happi", "ness"；（3）在词级和字符级之间取得平衡——高频词保留完整，低频词拆分为有意义的子词。GPT 系列用 BPE，BERT 用 WordPiece（类似但基于似然选择合并对）。

### 5. 预训练-微调范式的原理和优势？

预训练阶段在海量无标注文本上学习通用语言表示（MLM、因果LM等），微调阶段用少量标注数据适配下游任务。优势：（1）**标注效率**——预训练利用了互联网的海量文本，微调只需少量标注即可达到好效果（从几万降到几百）；（2）**通用表示**——预训练学到的表示对多种任务都有用，避免每个任务从零训练；（3）**少样本能力**——好的预训练模型即使5-10个样本也能取得合理效果。Prompt Tuning / LoRA 等参数高效微调方法进一步降低了微调成本。

### 6. 如何处理 NLP 中的中文分词问题？

深度学习时代推荐直接用字符级输入 + 子词分词，跳过传统分词。原因：（1）分词本身有误差，错误会传播到下游；（2）预训练模型（如 BERT）的子词分词器已经处理了中文的字符切分，无需额外分词；（3）字符级输入保留了最大信息量。但如果使用传统方法（TF-IDF + 机器学习），中文分词是必须的，jieba 是默认选择。对于领域文本（医学、法律），pkuseg + 自定义词典效果更好。中文预训练模型推荐 hfl/chinese-roberta-wwm-ext（全词遮蔽版，比字级遮蔽更好）。

### 7. 文本分类中如何处理类别不平衡？

四种策略：（1）**重采样**——对少数类过采样或多数类欠采样，简单有效但可能过拟合或丢信息；（2）**class_weight**——训练时给少数类更高权重，`CrossEntropyLoss(weight=...)` 一行搞定；（3）**Focal Loss**——自动降低易分类样本的权重，让模型关注难分类的少数类；（4）**数据增强**——对少数类文本做同义词替换、回译、EDA（随机删除/交换/替换），增加少数类样本。评估时不能只看准确率，应看 F1（特别是少数类的 F1）和 AUC-PR。

### 8. BLEU 和 ROUGE 的区别？

BLEU 衡量生成文本与参考文本的 n-gram 精确率（precision），关注生成的内容有多少是正确的，主要用于机器翻译。ROUGE 衡量 n-gram 召回率（recall），关注参考文本的内容有多少被生成覆盖，主要用于文本摘要。BLEU 高意味着生成少有错误但可能遗漏内容，ROUGE 高意味着覆盖全面但可能有冗余。两者都基于 n-gram 匹配，无法衡量语义等价性。BERTScore 基于预训练模型的语义嵌入计算相似度，是更好的语义级指标但计算成本更高。
