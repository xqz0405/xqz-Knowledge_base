# CSS选择器与优先级

## What — 是什么

> CSS 选择器决定样式应用到哪些元素，优先级（Specificity）决定冲突时哪条规则生效。

**核心概念：**

- **优先级计算**：`(ID, Class, Element)` 三位权重
  - `#id` → (1,0,0) = 100
  - `.class` / `[attr]` / `:hover` → (0,1,0) = 10
  - `div` / `::before` → (0,0,1) = 1
- **`!important`**：覆盖所有优先级（最后手段）
- **内联样式**：`style=""` 权重高于所有选择器
- **同优先级**：后声明的覆盖先声明的（源码顺序）

**关键特性：**

- 通配符 `*` 优先级为 (0,0,0)
- `:not()` / `:is()` / `:where()` 本身不计权重，参数计
- `:where()` 优先级强制为 0
- `@layer` 可将样式降级到低优先级层

## Why — 为什么

**适用场景：**

- 理解样式覆盖规则，调试 CSS 冲突
- 设计可维护的 CSS 架构（避免 !important 泛滥）
- 编写可预测的组件样式

**对比替代方案：**

| 维度 | CSS 优先级 | CSS-in-JS | Tailwind |
|------|-----------|-----------|----------|
| 可预测性 | 中（需理解权重） | 高（作用域隔离） | 高（原子化无冲突） |
| 调试 | 中 | 中 | 容易 |
| 学习成本 | 低 | 中 | 中 |

**优缺点：**

- ✅ 优点：
  - 原生机制，无额外依赖
  - 精确控制样式层级
- ❌ 缺点：
  - 优先级计算容易出错
  - 嵌套选择器权重累积，后期难以覆盖
  - `!important` 滥用导致维护困难

## How — 怎么用

### 快速上手

```css
/* 优先级从低到高 */
*              {}  /* (0,0,0) */
li             {}  /* (0,0,1) */
ul li          {}  /* (0,0,2) */
.item          {}  /* (0,1,0) */
ul .item       {}  /* (0,1,1) */
#nav           {}  /* (1,0,0) */
#nav .item     {}  /* (1,1,0) */
style="..."        /* 内联，最高 */
!important         /* 终极覆盖 */
```

### 代码示例

**`:where()` 降权：**

```css
/* :where() 优先级为 0，方便被覆盖 */
:where(.card) {
    padding: 16px;
    border-radius: 8px;
}

/* 任何选择器都能轻松覆盖 */
.card { padding: 24px; } /* (0,1,0) > (0,0,0) 生效 */
```

**`@layer` 分层：**

```css
/* 低优先级层 */
@layer base {
    a { color: blue; }
}

/* 高优先级层（后声明的层优先级更高） */
@layer components {
    a { color: green; }
}

/* 未分层的样式优先级最高 */
a { color: red; } /* 最终生效 */
```

**BEM 命名避免权重问题：**

```css
/* ❌ 嵌套选择器，权重累积 */
.nav .nav-item .nav-link { color: red; }

/* ✅ BEM：单 class，权重一致 */
.nav__link { color: red; }
.nav__link--active { color: blue; } /* 轻松覆盖 */
```

### 常见问题与踩坑

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 样式不生效 | 被更高优先级覆盖 | DevTools 查看被划线的规则，找到覆盖源 |
| !important 泛滥 | 优先级失控 | 用 BEM/CSS Modules 从根本解决，`:where()` 降权 |
| 第三方库样式难覆盖 | 库用了高优先级选择器 | 用更具体选择器或 `:where()` 反向操作 |
| `:not()` 权重计算 | `:not(.a)` 权重是 (0,1,0) | 把 `:not()` 的参数权重加上去 |

### 最佳实践

- 选择器嵌套不超过 3 层
- 用 BEM / CSS Modules 控制权重在 (0,1,0)
- 永远不要用 `!important`（除了覆盖第三方库）
- 用 `@layer` 管理全局样式层级

## 面试题

**Q1: CSS优先级如何计算？**
> 优先级按(ID, Class, Element)三位权重比较：#id为(1,0,0)，.class/属性选择器/伪类为(0,1,0)，元素选择器/伪元素为(0,0,1)。同权重时后声明覆盖先声明，内联样式优先级最高，!important覆盖一切。

**Q2: !important的使用原则是什么？**
> !important应作为最后手段，仅用于覆盖第三方库样式。滥用会导致样式无法被正常覆盖，形成优先级军备竞赛。应通过BEM命名、CSS Modules、@layer等机制从源头控制权重，而非依赖!important。

**Q3: :where()和:is()的区别是什么？**
> 两者功能相似，都接受选择器列表作为参数。关键区别在优先级：:where()优先级强制为0（方便被覆盖），:is()取参数中最高优先级。:where()适合基础样式定义，:is()适合简化选择器写法同时保留权重。

**Q4: 选择器嵌套过深有什么问题？**
> 嵌套过深导致权重累积（如`.nav .nav-item .nav-link`为(0,3,0)），后期难以覆盖；增加CSS文件体积；降低浏览器匹配性能（选择器从右向左匹配，每增加一层多一轮遍历）。建议嵌套不超过3层，用BEM单class方案控制权重。
